üî∞ Basic DevOps Questions
Q1What is DevOps and why is it important? +

Answer: DevOps is a cultural philosophy and set of practices that combines software development (Dev) and IT operations (Ops) to shorten the development lifecycle and provide continuous delivery with high software quality.

Key Benefits:

  • Faster deployment cycles
  • Improved collaboration between teams
  • Higher quality software releases
  • Better customer satisfaction
  • Reduced failure rates of new releases
Q2What are the key principles of DevOps? +

Answer: The core principles of DevOps include:

  • Automation: Automate repetitive tasks and processes
  • Continuous Integration: Regularly merge code changes
  • Continuous Delivery: Automate deployment pipeline
  • Infrastructure as Code: Manage infrastructure through code
  • Monitoring and Logging: Continuous monitoring of applications
  • Collaboration: Break down silos between teams
  • Feedback: Quick feedback loops for improvement
Q3Explain the difference between Continuous Integration, Continuous Delivery, and Continuous Deployment. +

Answer:

  • Continuous Integration (CI): Developers regularly merge code changes into a central repository, triggering automated builds and tests
  • Continuous Delivery (CD): Extends CI by ensuring code changes are automatically prepared for production release, but deployment requires manual approval
  • Continuous Deployment: Fully automated process where every code change that passes tests is automatically deployed to production without manual intervention
üîÑ CI/CD Pipeline Questions
Q4What are the stages of a typical CI/CD pipeline? +

Answer: A typical CI/CD pipeline includes:

  1. Source: Code committed to version control
  2. Build: Compile and package the application
  3. Test: Run automated tests (unit, integration, security)
  4. Package: Create deployable artifacts
  5. Deploy to Staging: Deploy to staging environment
  6. Test in Staging: Run acceptance and performance tests
  7. Deploy to Production: Deploy to production environment
  8. Monitor: Monitor application performance and health
Q5How do you implement Blue-Green deployment? +

Answer: Blue-Green deployment is a strategy that reduces downtime and risk by maintaining two identical production environments:

  • Blue Environment: Current live production environment serving all traffic
  • Green Environment: Identical environment where new version is deployed
  • Process: Deploy to Green ‚Üí Test thoroughly ‚Üí Switch traffic from Blue to Green ‚Üí Keep Blue as rollback option
  • Benefits: Zero downtime, instant rollback, reduced risk
üê≥ Docker & Containerization
Q6What is Docker and how does it differ from Virtual Machines? +

Answer: Docker is a containerization platform that packages applications with their dependencies.

Docker vs Virtual Machines:

  • Resource Usage: Containers share host OS kernel, VMs require separate OS instances
  • Performance: Containers have less overhead, faster startup times
  • Isolation: VMs provide stronger isolation, containers share kernel
  • Portability: Containers are more portable across environments
  • Size: Container images are smaller than VM images
Q7Explain Docker architecture and its components. +

Answer: Docker architecture consists of:

  • Docker Client: Command-line interface for user interaction
  • Docker Daemon: Background service managing containers, images, networks
  • Docker Images: Read-only templates for creating containers
  • Docker Containers: Running instances of Docker images
  • Docker Registry: Storage and distribution of Docker images (Docker Hub)
  • Dockerfile: Text file with instructions to build images
Q8What are Docker volumes and why are they important? +

Answer: Docker volumes provide persistent data storage for containers.

Types of volumes:

  • Named volumes: docker volume create myvolume
  • Host volumes: Mount host directory to container
  • Anonymous volumes: Temporary volumes created automatically

Importance: Data persistence, sharing data between containers, backup and restore capabilities

‚ò∏Ô∏è Kubernetes & Orchestration
Q9What is Kubernetes and what problems does it solve? +

Answer: Kubernetes is an open-source container orchestration platform that automates deployment, scaling, and management of containerized applications.

Problems it solves:

  • Container orchestration across multiple hosts
  • Automatic scaling based on demand
  • Service discovery and load balancing
  • Rolling updates and rollbacks
  • Resource management and allocation
  • Self-healing capabilities
Q10Explain Kubernetes architecture and key components. +

Answer: Kubernetes architecture consists of:

Master Node Components:

  • API Server: Central management component
  • etcd: Distributed key-value store
  • Controller Manager: Manages various controllers
  • Scheduler: Assigns pods to nodes

Worker Node Components:

  • kubelet: Node agent communicating with master
  • kube-proxy: Network proxy for services
  • Container Runtime: Runs containers (Docker, containerd)
‚òÅÔ∏è AWS & Cloud DevOps
Q11What are the key AWS services used in DevOps? +

Answer: Key AWS DevOps services include:

  • CodeCommit: Git-based source control service
  • CodeBuild: Fully managed continuous integration service
  • CodeDeploy: Automated application deployment service
  • CodePipeline: Continuous integration and delivery service
  • CloudFormation: Infrastructure as Code service
  • ECS/EKS: Container orchestration services
  • Lambda: Serverless compute service
  • CloudWatch: Monitoring and logging service
Q12How do you implement Infrastructure as Code using AWS CloudFormation? +

Answer: CloudFormation allows you to define AWS infrastructure using JSON or YAML templates.

Key concepts:

  • Templates: Define resources and their properties
  • Stacks: Collection of resources managed as a single unit
  • Parameters: Input values for templates
  • Outputs: Return values from stack
  • Mappings: Static lookup tables
  • Conditions: Control resource creation based on parameters

Benefits: Version control, repeatability, consistency, rollback capabilities

‚öôÔ∏è Ansible & Configuration Management
Q13What is Ansible and how does it work? +

Answer: Ansible is an agentless configuration management tool that automates software provisioning, configuration management, and application deployment.

Key features:

  • Agentless: Uses SSH for communication, no agents required
  • Idempotent: Running same playbook multiple times produces same result
  • YAML-based: Human-readable playbook syntax
  • Push-based: Control machine pushes configurations
  • Modules: Pre-built units for specific tasks
Q14Explain Ansible architecture and components. +

Answer: Ansible architecture includes:

  • Control Node: Machine where Ansible is installed
  • Managed Nodes: Target machines managed by Ansible
  • Inventory: List of managed nodes
  • Playbooks: YAML files defining automation tasks
  • Modules: Units of work executed by Ansible
  • Tasks: Individual actions performed by modules
  • Roles: Reusable automation content
  • Handlers: Special tasks triggered by notifications
üîß Jenkins & Automation
Q15What is Jenkins and what are its key features? +

Answer: Jenkins is an open-source automation server for continuous integration and continuous delivery.

Key features:

  • Extensible: 1000+ plugins available
  • Distributed builds: Master-slave architecture
  • Pipeline as Code: Jenkinsfile for version-controlled pipelines
  • Easy configuration: Web-based configuration interface
  • Integration: Supports various tools and technologies
  • Open source: Free and community-driven
Q16Explain Jenkins Pipeline and difference between Declarative and Scripted pipelines. +

Answer: Jenkins Pipeline is a suite of plugins for implementing continuous delivery pipelines as code.

Declarative Pipeline:

  • Structured syntax with predefined sections
  • Easier to read and write
  • Better validation and error handling
  • Uses pipeline block with agent, stages, steps

Scripted Pipeline:

  • Groovy-based programming approach
  • More flexibility and control
  • Uses node blocks
  • Better for complex logic and conditionals
üìä Monitoring & Logging
Q17Why is monitoring important in DevOps and what should be monitored? +

Answer: Monitoring is crucial for maintaining system reliability, performance, and user experience in DevOps environments.

What to monitor:

  • Infrastructure: CPU, memory, disk, network metrics
  • Application: Response times, error rates, throughput
  • Business: User activity, conversion rates, revenue
  • Security: Failed logins, suspicious activities
  • Logs: Application logs, system logs, security logs

Benefits: Early problem detection, performance optimization, capacity planning, improved user experience

Q18What is the ELK Stack and how is it used for logging? +

Answer: ELK Stack is a collection of three open-source tools for centralized logging and log analysis.

Components:

  • Elasticsearch: Distributed search and analytics engine for storing and indexing logs
  • Logstash: Data processing pipeline that ingests, transforms, and sends logs to Elasticsearch
  • Kibana: Visualization platform for creating dashboards and analyzing log data

Workflow: Applications ‚Üí Logstash (collect/process) ‚Üí Elasticsearch (store/index) ‚Üí Kibana (visualize/analyze)

Benefits: Centralized logging, real-time analysis, powerful search capabilities, custom dashboards

Q19Explain Prometheus and Grafana for monitoring. +

Answer: Prometheus and Grafana form a powerful monitoring stack.

Prometheus:

  • Time-series database for metrics collection
  • Pull-based monitoring model
  • PromQL query language
  • Built-in alerting capabilities
  • Service discovery for dynamic environments

Grafana:

  • Visualization and dashboarding platform
  • Supports multiple data sources
  • Rich set of visualization options
  • Alerting and notification features
  • Template variables for dynamic dashboards
Q20What are the different types of testing in DevOps pipeline? +

Answer: DevOps pipelines incorporate various types of automated testing:

  • Unit Tests: Test individual components or functions
  • Integration Tests: Test interaction between components
  • API Tests: Test application programming interfaces
  • Security Tests: Scan for vulnerabilities and security issues
  • Performance Tests: Test application under load conditions
  • Smoke Tests: Basic tests to verify core functionality
  • Regression Tests: Ensure new changes don't break existing features
  • Acceptance Tests: Validate business requirements
